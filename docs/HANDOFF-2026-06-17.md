## 2) Auth / Primeiro acesso — RESOLVIDO E VERIFICADO EM PROD

**Bug:** o link de acesso (recovery) já é sessão logada; o usuário caía no 2FA e
**entrava sem nunca criar senha** (sem como logar depois). Causa-raiz: o
middleware `src/proxy.ts` tratava `/update-password` como rota de auth e
redirecionava usuário autenticado → `/dashboard` → portão 2FA.

**Fix (commits `5129441` + `f958a19`, migration `0021`):**
- Flag `profiles.password_set_at` (NULL = ainda não criou senha). Backfill dos
  existentes (não afeta quem já usa).
- Portão do app (`(app)/layout.tsx`): 1º acesso (password_set_at NULL) → força
  `/update-password` ANTES do 2FA (o link já provou posse do e-mail).
- `updatePasswordAction` grava `password_set_at` + cookie 2FA → entra direto.
- Middleware: `/update-password` saiu dos `authPrefixes` (sessão recovery precisa
  acessar).
- `/update-password` mostra copy de 1º acesso ("Crie sua senha de acesso").
- **Verificado em prod** por curl (gera link recovery → segue cadeia): cai em
  "Criar senha", sem loop, sem 2FA. ✅

**Login recorrente / esqueci-senha:** continuam passando pelo 2FA por e-mail
(mantido em TODO login por ora — Francis decide depois se afrouxa).

## 3) Perf — consolidação de auth (commit `2896518`)

O portão fazia ~5 `getClaims()` + 4 queries de profile por página. Consolidado pra
**2 round-trips**: `getSessionClaims()` (getClaims local, verifica JWT sem rede) +
`getGateProfile()` (1 SELECT com todas as colunas de gate), ambas memoizadas com
React `cache()`. As 5 funções (getCurrentUser/getAccessGate/getAccessExpiry/
needsOnboarding/needsPasswordSetup/getProfileDetails) leem desse cache. Mesma
segurança (defesa em camadas: middleware + RSC + RLS), menos idas ao banco.

## Pendências / decisões aguardando

- **2FA — e-mail lento (Brevo Free):** não é bug; é latência do IP compartilhado
  do tier free (cai em spam/demora). Pagar Brevo (~US$9) **não acelera** (só tira
  limite de 300/dia, que não estouram). Recomendação pesquisada: (a) conferir
  **SPF/DMARC** do domínio (DKIM já tem; não consegui checar DNS daqui — sandbox
  sem porta 53); (b) se quiser 2FA rápido/confiável, **trocar provider**: Resend
  (nativo Next.js, grátis 3k/mês, migração = 1 função) ou Postmark (mais rápido,
  <1s, US$15/mês). **Aguardando Francis decidir.** A migração no código é pequena
  (só `src/lib/email/brevo.ts` → `sendTransactionalEmail`).
